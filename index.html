Понятия Git

HEAD – указатель ссылающийся на фиксированную точку (commit) ветки, который станет родителем следующей фиксированной точки.

Переход в папку для работы с Git

  1. Перейти к командной строке Windows или bash Linux;
  2. git --version показывает установленную версию Git;
  3. Перейти в папку с проектом cd <путь_к_папке>
  4. code . запустить редактор кода Visual Studio Code.

init инициализация проекта

git init – создаёт в текущем каталоге новый подкаталог с именем .git с настройками Git проекта.
git status – показывает статус Git, в последствии перед выполнением команд лучше проверять статус для выбора правилных действий.
On branch master – ветка master включена.
No commits yet – пока нет коммитов.
Untracked files: – без отслеживания.

add начало отслеживания файлов

Приступая к редактированию проекта нужно выбрать что нам необходимо отслеживать.
git add <имя_файла или имя_папки/> – начать отслеживать файл имя_файла или имя_папки/.
git add . – начать отслеживать все файлы (точка через пробел в конце).

commit фиксирование состояния ветки (branch)

При необходимости создаем запись состояния ветки.
git commit -m <"имя_коммита"> – создает точку сохранения состояния ветки.

log показывает историю фиксаций

git log – показывает историю созданных commit снизу вверх.
git log -p – папаметр -p показывает внесенные изменения фиксированных точек.
Прокрутка стрелочками вниз и вверх.
Записать log в файл кнопка s.
Что бы продолжить работу кнопка q.
git log --oneline – папаметр --oneline выводит краткую запись фиксированных точек.

checkout работа с фиксированными точками проекта

git checkout <хеш_фиксированной_точки> – переходит на указанную фиксированную точку.

reset работа с фиксированными точками проекта

git reset [--soft] <хеш_фиксированной_точки> – не перемещяет HEAD, не меняет файлы.
git reset [--mixed] <хеш_фиксированной_точки> – параметр --mixed по умолчанию, перемещяет HEAD, не меняет файлы.
git reset [--hard] <хеш_фиксированной_точки> – перемещяет HEAD, меняет файлы.

revert работа с фиксированными точками проекта

git revert [--edit] <хеш_фиксированной_точки> – параметр --edit по умолчанию, берет состояние фиксированной точки и создает новую.
git revert – не перемещяет HEAD, не меняет файлы.
--no-edit – не оставляет комменарий.
--no-commit – не создает фиксированную точку, обращенные изменения будут добавлены в раздел проиндексированных файлов и рабочий каталог.

branch работа с ветками проекта

git branch – показывает ветки проекта, * обозначается выбранная ветка.
git branch [-r] – показывает ветки из удаленного репозитория.
git branch <имя_новой_ветки> – создает ветку имя_новой_ветки, копию выбранной ветки.
git branch -D <имя_ветки> – параметр -D удаляет ветку с указанным именем.
git branch -m <имя_ветки> – параметр -m переименовывает ветку.

checkout работа с ветками проекта

git checkout <имя_ветки> – переключается на имя_ветки.
git checkout -b <имя_ветки> – создает ветку и переключается на нее.

merge слияние веток

git merge <имя_ветки> – объединяет текущию ветку с имя_ветки.

remote регитрация нового удаленного репозитория

git remote add <origin> <адрес_репозитория> – регистрирует (origin - название удаленного репозитория) по адрес_репозитория.

remote просмотр удаленных репозиториев

git remote – показывает все удаленные репозитории с которыми работаем.
git remote show <origin> – показывает информацию (origin - название удаленного репозитория).
git remote -v – показывает все удаленные репозитории вместе с адресом репозитория.

remote переименование удаленных репозиториев

git remote rename <old_name> <new_name> – переименовывает удаленный реполиторий с именем old_name на новое имя new_name.

remote удаление удаленных репозиториев

git remote rm <origin> – удаляет удаленный репозиторий (origin - название удаленного репозитория).

push публиция файлов на удаленном сервере

git push -u <origin> <имя_ветки> – загружает в (origin - название удаленного репозитория) указанную ветку (имя_ветки).

clone клонирование репозитория

git clone <адрес_репозитория> ./ – загружает удаленный репозиторий в пустую папку, команда дожна выполнятся из выбранной папки.

pull обновление из удаленного репозитория

git pull <адрес_репозитория> – обновляет активную ветку из удаленного репозитория (сохранить ожидающие изменения в ветке).

fetch обновление из удаленного репозитория

git fetch <адрес_репозитория> – извлекает все ветки из репозитория (создаются отдельные ветки).
git fetch <адрес_репозитория> <имя_ветки> – извлекает указанную ветку.
git fetch [--all] – извлекает все зарегистрированные удаленные репозитории и их ветки.
git fetch origin – синхронизирует локальный репозиторий с главной веткой удаленного центрального репозитория, команда покажет загруженные ветки.

Git Tags

Tags указывают на важные фиксиорванные точки в истории Git.

Tag работа с tags проекта

git tag <имя_тэга> создает ярлык (teg) для фиксированной точки.
git tag -a <имя_тэга> -m "имя_комментария" – добавляет к тегу комментарий.
